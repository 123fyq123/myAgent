将文档写入 `APIDocs/<模块>/<功能目录>/README.md`。

可按实际场景裁剪，但尽量保留“链路总览”“关键文件定位”“详细调用链”“调试与验证”“注意点”。

```markdown

- 主模块：`user` / `project` / 其他
- 相关模块：`project-api`、`project-user`、`project-project`、`frontend` 等

## 链路总览

先用 4-8 行把整条链路讲清楚，建议格式类似：

1. 用户在前端什么位置触发
2. 前端调用哪个 HTTP 接口
3. `project-api` 哪个路由/handler 接收
4. API 层调用哪个 gRPC 方法或本地 service
5. 下游服务在哪个 service / repo 落地
6. 数据从哪里查或写回
7. 最终响应如何返回前端

| `backend/ms_project/project-user/...` | 业务实现 | 说明逻辑落点 |

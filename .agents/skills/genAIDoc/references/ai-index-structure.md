# AI_INDEX.md 结构规范

本文档定义 `AI_INDEX.md` 的结构模板。生成时按此填充。

## 位置与作用

- **位置**：仓库根目录、`backend/ms_project/` 等工作区根目录
- **作用**：菜单页，列出子模块定位指针

## 模板

```markdown
# AI_INDEX

## 作用
<索引页说明：这是哪一层索引，怎么用>

## 技术栈

### 开发语言
- <语言及版本，如 Go 1.18、Node 16、Python 3.10>

### 框架与运行时
- <框架及版本，如 Gin、Vue 2、Vuex、Vue Router、Ant Design Vue>

### 数据与中间件
- <数据库、缓存、消息队列、服务发现等，如 MySQL、Redis、Etcd、Kafka、Jaeger>

### 基础设施
- <容器与部署相关，如 Docker Compose、Nacos>

## 模块清单

### <模块1>
- 定位说明（一句话）
- 下一步看：`<模块1>/AI_CONTEXT.md`

### <模块2>
- ...

## 典型调用链
1. <步骤1>
2. <步骤2>

## 不要碰
- <运行时产物、生成代码等>
```

## 章节说明

| 章节 | 内容 | 必填 |
|---|---|---|
| 作用 | 索引页说明，怎么用 | 是 |
| 技术栈 | 开发语言、框架、数据库、中间件、基础设施 | 是 |
| 模块清单 | 子模块定位 + 指向对应 AI_CONTEXT.md | 是 |
| 典型调用链 | 跨模块的完整请求链路示例 | 按需 |
| 不要碰 | 列出不该碰的目录 | 按需 |

# AI_CONTEXT.md 结构规范

本文档定义 `AI_CONTEXT.md` 的结构模板。生成时按此填充。

## 位置与作用

- **位置**：模块根目录
- **作用**：模块详情说明书

## 模板

```markdown
# <模块名> 说明

## 模块定位
<一句话说明这个模块是什么、在整体架构里的位置>

## 主要职责
- 职责1
- 职责2

## 目录结构

### main.go / 入口文件
- 启动步骤说明（按实际执行顺序列出）

### <关键目录1>/
- 文件清单 + 作用说明

### <关键目录2>/
- ...

## 上下游关系
- 上游：<调用方>
- 下游：<被调用方>
- 共享依赖：<公共模块>

## 何时先看这个模块
- 场景1
- 场景2

## 不要碰
- 运行时产物、生成代码等
```

## 章节说明

| 章节 | 内容 | 必填 |
|---|---|---|
| 模块定位 | 一句话说明模块在架构里的位置 | 是 |
| 主要职责 | 列出该模块负责的业务或能力 | 是 |
| 目录结构 | 列出关键目录和文件，标注作用 | 是 |
| 上下游关系 | 上游调用方、下游依赖、共享依赖 | 是 |
| 何时先看这个模块 | 列出适合优先查该模块的场景 | 是 |
| 不要碰 | 列出运行时产物、生成代码等不该碰的目录 | 按需 |

# Faber AI 测试功能清单

> 使用方法：完成一项就在对应项前勾选。带“需配置”的项目依赖模型 API、SMTP 或外部服务；基础版未配置这些服务时，可先跳过。

## 0. 测试前置条件

- [ ] Docker 服务已启动：PostgreSQL、Redis、Elasticsearch、Milvus、etcd、MinIO。
- [ ] PostgreSQL 可连接：`localhost:15432`，数据库为 `faber-ai`。
- [ ] 已导入与当前后端代码匹配的数据库结构，且 `users`、`agents`、`agent_skills`、`skills` 等表存在。
- [ ] 后端从 `backend/app` 目录以 `go run .` 启动，访问 `GET /health` 返回成功。
- [ ] 前端在 `frontend` 目录以 `corepack pnpm dev` 启动，可打开 `http://127.0.0.1:5173`。
- [ ] 已准备可登录的基础账号：`admin / 123456`。

## 1. 账号与访问控制

- [ ] 未登录访问首页、智能体、知识库等受保护页面时，自动跳转登录页。
- [ ] 使用 `admin / 123456` 可以成功登录并进入首页。
- [ ] 登录后刷新浏览器，登录状态仍能正确恢复或按预期要求重新登录。
- [ ] 输入错误账号或密码时，页面有明确错误提示，且不会登录成功。
- [ ] 新用户注册可以创建账号。（需 SMTP；基础版可跳过邮箱验证流程）
- [ ] 邮箱验证链接可完成账号验证。（需 SMTP）
- [ ] 忘记密码、验证码校验、重置密码流程可用。（需 SMTP）
- [ ] 无管理员权限的账号访问管理员页面或接口时被拒绝。
- [ ] 个人资料页可查看并更新允许修改的个人资料。

## 2. 模型与供应商配置

- [ ] 可以新建模型供应商配置（名称、地址、API Key 等）。
- [ ] 供应商配置列表能正确展示已保存项。
- [ ] 可以新建并查看聊天模型配置。
- [ ] 可以新建并查看 Embedding（向量化）模型配置。
- [ ] 模型 API Key 缺失或无效时，前端/接口能给出可理解的失败提示。
- [ ] 使用有效模型配置可完成一次最小对话测试。（需模型 API Key）
- [ ] 使用有效 Embedding 模型可完成一次文本向量化。（需模型 API Key）

## 3. 智能体管理

- [ ] 可创建基础智能体：名称、描述、系统提示词、模型供应商和模型名称。
- [ ] 创建后，智能体列表能显示该智能体。
- [ ] 可进入智能体详情并读取完整配置。
- [ ] 可修改智能体名称、提示词、模型参数、可见性和状态。
- [ ] 可删除智能体，删除后列表及详情均不再显示。
- [ ] 可为智能体批量绑定工具，并可单独解绑工具。
- [ ] 可为智能体绑定知识库，并可解除绑定。
- [ ] 可为智能体绑定工作流，并可解除绑定。
- [ ] 可为智能体绑定 Skill，并可解除绑定。（需 `skills`、`agent_skills` 表）
- [ ] 创建智能体时不会出现 `agents.agent_mode` 或 `agents.deep_config` 字段缺失错误。

## 4. 智能体对话与上下文

- [ ] 可为指定智能体创建新的聊天会话。
- [ ] 会话列表能显示已创建会话。
- [ ] 发送普通文本消息后，可以获得智能体回复。（需聊天模型 API Key）
- [ ] 流式回复可以完整显示，加载状态和异常状态正常。
- [ ] 同一会话连续提问时，智能体能读取此前消息上下文。（需聊天模型 API Key）
- [ ] 会话消息列表能按正确顺序展示用户和智能体消息。
- [ ] 删除会话后，列表和消息查询均不再返回该会话。
- [ ] 模型未配置、超时或调用失败时，不会导致前端页面卡死或后端崩溃。

## 5. 知识库与文档

- [ ] 可创建知识库并在列表中查看。
- [ ] 可查看知识库详情并修改名称、描述或配置。
- [ ] 可上传 Markdown、TXT、PDF、DOCX、HTML、EPUB 等支持格式的文档。
- [ ] 上传后能看到文档列表、处理状态和基础元数据。
- [ ] 文档解析、分块和向量入库完成后，状态显示成功。（需 Embedding 模型与 Milvus）
- [ ] 在知识库详情页中可执行检索，并返回相关文本片段。（需 Embedding 模型与 Milvus）
- [ ] 删除单个文档后，文档记录及对应向量索引被移除。
- [ ] 删除知识库后，知识库、文档及对应向量索引被移除。
- [ ] 智能体绑定知识库后，对话能引用相关知识库内容。（需聊天与 Embedding 模型）
- [ ] 上传不支持格式、空文件或超大文件时，有清晰的校验提示。

## 6. 工具与 MCP

- [ ] 可创建工具，并在工具列表中查看。
- [ ] 可编辑和删除工具。
- [ ] 可对工具执行测试，成功时展示调用结果。
- [ ] 工具地址无效、参数不合法或服务不可达时，能显示可理解的错误信息。
- [ ] 可查询 MCP 服务下的工具列表。（需 MCP 服务可用）
- [ ] 智能体绑定工具后，对话中可按预期调用工具。（需聊天模型及工具服务可用）
- [ ] 未配置高德等外部 API Key 时，基础应用仍能启动；相关外部工具不应被错误注册。

## 7. 工作流与节点

- [ ] 可创建工作流，并在工作流列表中查看。
- [ ] 可打开工作流编辑器并加载已保存的节点和连线。
- [ ] 可新增、编辑、删除节点并保存工作流。
- [ ] 保存后重新打开，节点参数、位置和连线保持一致。
- [ ] 可获取工作流详情。
- [ ] 可执行工作流并查看执行结果。（部分节点可能需模型或外部服务）
- [ ] 节点测试接口可正确反馈节点配置是否有效。
- [ ] 可删除工作流，且已绑定该工作流的智能体不再使用失效配置。

## 8. Skills 与来源管理

- [ ] 可创建、编辑、删除 Skill。
- [ ] Skill 列表和全部 Skill 列表均可正确返回。
- [ ] 可安装 Skill，安装失败时有明确原因提示。
- [ ] 可新增、编辑、删除 GitHub Skill 来源。
- [ ] 可查看单个 GitHub 来源详情和来源列表。
- [ ] GitHub 地址不可访问或仓库结构不符合要求时，不会影响其他功能。（需网络访问）

## 9. 市场、订阅与云存储

- [ ] MCP 市场页面可以加载。
- [ ] Agent 市场列表可以加载、保存 Agent Card，并删除市场项。（需 A2A 服务）
- [ ] 订阅页面可以读取当前用户订阅信息。
- [ ] 云存储页面可正常进入；已配置存储服务时可执行上传、查看或管理操作。
- [ ] 未配置订阅、A2A 或云存储服务时，页面能明确提示配置缺失，而非报错白屏。

## 10. 数据、异常与回归

- [ ] PostgreSQL 中登录用户、智能体、会话、消息、知识库和文档数据与页面操作一致。
- [ ] Redis、Milvus、Elasticsearch 任一依赖不可用时，后端日志能定位问题，接口返回合理失败信息。
- [ ] 重启 Docker 容器后，已有 PostgreSQL 数据仍存在。
- [ ] 重启后端后，已保存的账号、智能体、知识库和工作流仍可读取。
- [ ] 重启前端后，已保存的数据不丢失，路由访问正常。
- [ ] 前端控制台无持续性未处理异常；后端无 panic。
- [ ] 对不存在的资源 ID、非法 UUID、缺少必填参数等请求，接口返回合理的 4xx 错误。
- [ ] 删除资源后刷新页面或再次请求，不会出现残留数据或服务器错误。

## 11. 当前环境的重点回归项

- [ ] 登录 `admin / 123456` 成功。
- [ ] 创建智能体不再出现 `agent_mode`、`deep_config` 字段缺失错误。
- [ ] 绑定 Skill 不再出现 `agent_skills` 表不存在错误。
- [ ] 注册流程若未配置 SMTP，不影响已有 `admin` 账号的登录和基础功能测试。
- [ ] 未配置模型 API Key 时，可测试账号、智能体 CRUD、工作流 CRUD、知识库 CRUD 等不依赖模型的功能；对话和向量检索应显示配置缺失提示。

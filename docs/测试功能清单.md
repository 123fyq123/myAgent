# Faber AI 项目测试功能清单

> 使用方式：测试通过就在前面勾选。标注“需配置”的项目依赖模型 API Key、SMTP、GitHub、A2A 或云存储等外部服务；当前先跑基础版时，可以先跳过这些项。

## 0. 环境与启动

- [ ] Docker Desktop 已启动。
- [ ] 后端依赖容器已启动：PostgreSQL、Redis、Milvus、etcd、MinIO。
- [ ] PostgreSQL 可用：Host `localhost`，Port `15432`，Database `faber-ai`，User `mszluai`。
- [ ] 数据库已导入基础 SQL，能看到 `users`、`agents`、`knowledge_bases`、`documents`、`workflows`、`tools` 等表。
- [ ] 后端从 `backend/app` 目录启动，避免找不到 `etc/config.yml`。
- [ ] 访问 `GET http://127.0.0.1:8888/health` 返回成功。
- [ ] 前端依赖已安装完成：`frontend/node_modules` 存在。
- [ ] 前端从 `frontend` 目录启动，页面可访问 `http://127.0.0.1:5173`。
- [ ] 前端代理 `/api` 能正常转发到后端 `8888` 端口。
- [ ] 准备好基础测试账号：`admin / 123456`。

## 1. 登录、注册与账号

- [ ] 打开前端后，未登录状态会跳转到登录页。

- [ ] 登录成功后能进入首页/控制台。

- [ ] 密码错误时，页面有明确错误提示。

- [ ] 注册接口可以创建新用户。
- [ ] 注册后不会因为 SMTP 未配置导致后端崩溃。
- [ ] 邮箱验证流程可用。（需配置 SMTP）
- [ ] 忘记密码、验证码、重置密码流程可用。（需配置 SMTP）

# benchmark 说明

## 模块定位
`benchmark` 是后端压测辅助模块，用于对智能体聊天接口发起并发 HTTP 请求。

## 主要职责
- 提供命令行压测脚本。
- 构造智能体聊天请求体并统计响应。
- 用标准库并发能力模拟多请求场景。

## 目录结构

### main.go / 入口文件
- 本模块没有 `main.go` 文件。

### agent_chat_benchmark.go
- 使用 `flag` 读取压测参数。
- 使用 `net/http` 构造请求。
- 使用 `sync`、`sync/atomic` 和 `time` 统计并发结果。
- 请求体与响应处理逻辑集中在该文件内。

### go.mod
- `go.mod`：模块名 `benchmark`，该文件当前没有显式第三方依赖。

## 上下游关系
- 上游：开发者在本地命令行运行压测脚本。
- 下游：请求 `app` 暴露的智能体聊天 HTTP 接口。
- 共享依赖：Go 标准库的 HTTP、并发和 JSON 能力。

## 何时先看这个模块
- 需要验证智能体聊天接口吞吐、延迟或并发错误率时。
- 调整压测请求体、认证头或目标地址时。

## 不要碰
- 不要把压测输出、临时结果或本地日志提交到源码目录。
- `go.mod` 应通过 Go 命令维护。

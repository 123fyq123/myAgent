# model 说明

## 模块定位
`model` 是跨业务包共享的 GORM 数据模型模块，定义数据库表映射、枚举及 JSON 字段序列化规则。

## 主要职责
- 定义用户、智能体、LLM、工具、订阅和知识库实体。
- 提供 JSON、字符串数组、MCP 配置、模型配置和参数 schema 的 `driver.Valuer` 与 `sql.Scanner` 实现。
- 提供 LLM 嵌入模型配置转换和默认智能体构造函数。

## 目录结构

### go.mod / 模块文件
- 声明 `model` 模块，源码使用 GORM、Eino schema、Eino 嵌入模型扩展和 Thunder AI 配置类型。

### base.go
- 定义带 UUID、创建时间和更新时间的 `BaseModel`，以及通用 JSON 字段类型。

### users.go
- 定义用户实体、状态枚举和用户传输对象。

### agents.go
- 定义智能体、智能体工具、智能体知识库关联、状态和可见性枚举，并构造默认智能体。

### llms.go
- 定义模型供应商配置、LLM 实体、模型类型与状态枚举，以及嵌入模型配置转换。

### tools.go
- 定义工具实体、工具类型、MCP 配置和 Eino 参数 schema 的数据库序列化逻辑。

### knowledges.go
- 定义知识库、文档、文档分块、存储类型和状态枚举，以及字符串数组 JSON 类型。

### subscriptions.go
- 定义订阅方案、支付时长、支付方式和订阅配置类型。

## 上下游关系
- 上游：`app` 的认证、智能体、LLM、工具和知识库业务包读写这些实体。
- 下游：依赖 PostgreSQL 驱动接口、GORM、UUID、Eino 和 Thunder AI 类型。
- 共享依赖：不依赖工作区内其他模块。

## 何时先看这个模块
- 需要新增或调整数据库表字段、枚举、关联关系或表名时。
- 需要排查 JSON 字段在 GORM 读写中的序列化和反序列化时。
- 需要理解智能体、工具、LLM 和知识库的持久化边界时。

## 不要碰
- `go.sum` 不存在于该模块；若后续生成，应由 Go 模块工具维护。
